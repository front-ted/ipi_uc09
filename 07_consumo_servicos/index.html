Consumo de serviços web: tipos, ferramentas de back-end e front-end, chamadas síncronas e assíncronas, envio de parâmetros e tratamento de respostas

Introdução

Neste conteúdo, você aprenderá os fundamentos essenciais para consumir serviços web de forma eficiente, explorando os diferentes tipos disponíveis, como REST (representational state transfer), além das ferramentas utilizadas no back-end e no front-end para conectar aplicações a serviços externos.

Você desenvolverá uma aplicação front-end utilizando HTML (hypertext markup language), JavaScript e a biblioteca jQuery, que consumirá a API (application programming interface) de notícias implementada no material sobre programação de serviços. Durante o processo, será criado um CRUD (create, read, update, delete) completo, incluindo funcionalidades para listar, cadastrar, alterar e excluir notícias.

Além disso, você compreenderá as diferenças entre chamadas síncronas e assíncronas, analisando como cada abordagem afeta o fluxo de execução das aplicações. Também será demonstrado o processo de envio de parâmetros e o tratamento de respostas, lidando com formatos como JSON (JavaScript object notation), garantindo a correta interpretação e manipulação dos dados recebidos.

Consumo de serviços web

O desenvolvimento front-end foca a criação e codificação dos elementos visuais de um site, ou seja, o “lado do cliente”, permitindo a interação do usuário. Já o back-end trabalha no “lado do servidor”, gerenciando serviços e garantindo que os elementos do front-end funcionem corretamente, como em sites de mídia social que armazenam informações de usuários em banco de dados, como Oracle, SQL Server, MySQL, entre outros.

A interação entre o front-end e os serviços do back-end é possível por meio de solicitações HTTP (hypertext transfer protocol). Funcionalidades como atualizar um usuário novo/antigo em um banco de dados, publicar uma postagem de blog, excluir uma imagem do seu perfil e atualizar sua biografia exigem uma interação entre um servidor e um cliente para modificar os dados. Para interagir com servidores no front-end, é possível usar diferentes métodos HTTP, como GET, POST, PUT e DELETE, para solicitar dados, conhecidos também por request ou requisições.

Tipos

Os serviços web são tecnologias fundamentais que permitem a comunicação entre sistemas distribuídos na internet. Por meio de APIs, os serviços web oferecem a capacidade de integrar diferentes sistemas, aplicativos e plataformas. Como visto nos conhecimentos sobre web services e REST, os principais tipos de serviços web são o SOAP (simple object access protocol) e o REST, cada um com suas características específicas e aplicabilidades.

Neste conteúdo, será adotado o REST, por ser uma abordagem mais leve e simples para serviços web. Baseado em métodos HTTP como GET, POST, PUT e DELETE, o REST utiliza URLs (uniform resource locator) para representar recursos e permite que os dados sejam trocados em diferentes formatos, com o JSON sendo o mais comum.

Ferramentas de back-end e front-end

Back-end

A camada back-end é responsável pelo processamento lógico, pela manipulação de dados e pela comunicação com banco de dados. Em sistemas dinâmicos, essas comunicações ocorrem de maneira constante, por exemplo: determinado usuário realiza o login em determinado sistema e os dados são tratados e comparados com os presentes na base de dados em questão. Uma vez verificado, o acesso à área restrita do sistema é liberado ou bloqueado. Como é possível perceber, é nessa camada também que são encontrados os mecanismos mais consistentes para a segurança de um sistema.

Nos exemplos no decorrer deste conteúdo, será utilizada a API de notícias implementada no material sobre programação de serviços, que foi desenvolvida com o Node.js e o framework Express.js, seguindo o protocolo REST. É fundamental que o serviço da API esteja iniciado; caso contrário, as requisições realizadas durante a construção da aplicação web não funcionarão e não produzirão os resultados esperados.

Front-end

A camada front-end é tudo o que está sendo apresentado na tela do usuário, ou seja, o que pode ser visualizado por ele ao acessar determinada página. É toda a parte visível de um site, como formulários, imagens, menus, botões, listas e tabelas. Ainda pertencente à camada do front-end, pode-se ressaltar toda a interatividade presente em uma página, havendo sites desenvolvidos apenas com a utilização dessa camada visual. Os chamados sites estáticos são aqueles em que não há interação com o banco de dados. As principais linguagens para o desenvolvimento front-end são: HTML, CSS e JavaScript.

Nesta seção, será implementada uma aplicação front-end utilizando HTML, JavaScript e a biblioteca jQuery, que consumirá a API de notícias.

A seguir, acompanhe o podcast sobre os fundamentos essenciais para consumir serviços web de forma eficiente, as diferenças essenciais entre front-end e back-end e como escolher a ferramenta ideal para a sua demanda.

Como realizar uma solicitação GET para listagem de notícias

Inicialmente, é preciso criar uma página em HTML com nome de index.html. Em um segundo momento, cria-se o arquivo js.js, que será responsável por realizar as requisições e tratar as respostas com o serviço de back-end da API de notícias.

No arquivo index.html, adiciona-se a biblioteca jQuery dentro da tag <head> do documento HTML. Além disso, no corpo da página <body>, há uma <div id="news"></div>, que será usada para listar as notícias.

Veja como ficou o código index.html:

                                        
<!DOCTYPE html> 
<html lang="en"> 
                                            
                                             
<head> 
    <meta charset="UTF-8" /> 
    <meta name="viewport" content="width=device-width, initial-scale=1.0" /> 
    <meta http-equiv="X-UA-Compatible" content="ie=edge" /> 
    <script src="https://code.jquery.com/jquery-3.4.1.js"></script>
    <title>Noticias</title> 
</head> 
                                            
    <body> 
        <div id="news"></div> 
        <script src="js.js"></script> 
    </body>                               
</html>

No arquivo js.js, será implementada a funcionalidade para carregar dinamicamente notícias da API de notícias, utilizando o método $.get() da biblioteca jQuery. Esse método permite enviar uma solicitação HTTP GET para uma URL específica e processar os dados retornados, atualizando a página sem necessidade de recarregá-la.

A lógica está encapsulada na função carregarNoticias, que realiza a solicitação GET para a URL http://localhost:3000/noticias. Esse método recebe dois parâmetros: a URL de destino e uma função de callback que é executada quando a requisição é bem-sucedida. Dentro dessa função de callback, os dados retornados pelo servidor (uma lista de notícias) são iterados com forEach. Para cada notícia, elementos HTML são criados dinamicamente, como <h3> para o título e <p> para o conteúdo, e adicionados ao elemento <div id="news"> na página por meio do método $('#news').append().

Antes de adicionar as notícias, o conteúdo preexistente de <div id="news"> é removido com $('#news').empty(), garantindo que a lista de notícias não seja duplicada caso a função seja chamada novamente. Essa função carregarNoticias é chamada automaticamente ao carregar a página, graças ao método $(document).ready().

Dessa forma, o código permite que as notícias sejam exibidas de forma dinâmica e atualizadas sem recarregar toda a página, proporcionando uma experiência mais fluida e interativa para o usuário. 

                                         
$(document).ready(function () { 

 
        // Função para carregar as notícias 
        function carregarNoticias() { 
            $.get("http://localhost:3000/noticias", function (data, status) { 
                $('#news').empty(); 
                $('#news').append('<h1>Listagem de Notícias</h1>'); 
                data.forEach(function (noticia) { 
                    $('#news').append( 
                        `<div id="noticia-${noticia.id}"> 
                            <h3>${noticia.titulo}</h3> 
                            <p>${noticia.conteudo}</p> 
                        </div>` 
                    ); 
                }); 
            }); 
        } 
                                                
        // Carregar as notícias ao carregar a página 
        carregarNoticias(); 
});

Como enviar uma solicitação POST para cadastrar uma nova notícia

Agora, você criará um arquivo chamado cadastro.html. Nele, haverá um formulário de cadastro de notícias com os campos título e conteúdo e a <div id="result"></div>, que receberá uma mensagem de confirmação de cadastro sem necessidade de atualizar a tela. 

                                         
<!DOCTYPE html> 
<html lang="en"> 
                                            
<head> 
    <meta charset="UTF-8" /> 
    <meta name="viewport" content="width=device-width, initial-scale=1.0" /> 
    <meta http-equiv="X-UA-Compatible" content="ie=edge" /> 
    <script src="https://code.jquery.com/jquery-3.4.1.js"></script> 
    <title>Notícias - Cadastrar</title> 
</head> 
                                            
<body> 
    <h1>Cadastro de notícias</h1> 
    <form method="post" action=""> 
        <label>Titulo: <input type="text" name="titulo" id="titulo"></label><br><br> 
        <label>Conteúdo: <textarea cols="50" name="conteudo" id="conteudo"></textarea></label><br><br> 
        <input type="submit" value="Cadastrar"> 
    </form> 
    <div id="result"></div> 
    <script src="js.js"></script> 
</body> 
                                            
</html>

Já no arquivo js.js, será adicionado dentro de $(document).ready() o código do método responsável por receber os dados do formulário de cadastro, enviar os dados via Ajax para um servidor back-end e tratar as respostas recebidas. O objetivo é evitar que a página seja recarregada e garantir uma comunicação dinâmica com o servidor. 

                                         
//cadastrar noticias 
$("form").submit(function (event) { 
                                                                               
        // Impede o envio normal do formulário 
        event.preventDefault(); 
                                            
        // Serializa os valores do formulário 
        const formValues = { 
            titulo: $('#titulo').val(), 
            conteudo: $('#conteudo').val() 
        }; 
                                            
        $.ajax({ 
            type: 'POST', 
            url: 'http://localhost:3000/noticias', 
            data: JSON.stringify(formValues), 
            contentType: 'application/json' 
        }) 
        .done(function (data) { 
            // enviar mensagem de registro salvo 
            $('#result').append("Registro salvo!"); 
        }) 
        .fail(function (msg) { 
            // caso a solicitacao de POST tenha falhado 
            alert("Falha no POST"); 
        }); 
})

Quando o formulário é enviado, o evento padrão de envio é impedido com event.preventDefault(). Em seguida, os valores dos campos de entrada do formulário (#titulo e #conteudo) são coletados usando os métodos $('#titulo').val() e $('#conteudo').val(), formando um objeto JSON chamado formValues. Esse objeto é convertido para uma string JSON com JSON.stringify() e enviado ao servidor por meio de uma requisição POST AJAX para a URL especificada.

A resposta do servidor é tratada com os métodos .done() e .fail(). Se a requisição for bem-sucedida, uma mensagem é adicionada à página indicando que o registro foi salvo. Caso ocorra algum erro, uma mensagem de falha é exibida ao usuário, alertando sobre o problema na solicitação. Essa abordagem proporciona uma interação mais fluida e moderna, sem a necessidade de recarregar a página inteira para cada envio de dados.

Implementando as funcionalidades de alterar e excluir notícias

Para expandir o exemplo e implementar as funcionalidades de edição e exclusão, completando assim as operações CRUD, será preciso seguir os seguintes passos:

Passo 1: no arquivo js.js, modifique a função carregarNoticias, adicione os botões de editar e excluir para cada notícia e implemente as funções correspondentes.

  • Adicione botões de editar e excluir, que chamam as funções editNews(id) e deleteNews(id), respectivamente, passando o ID da notícia.
  • Crie as funções deleteNews e editNews.
                                         
$(document).ready(function () { 

        // Função para carregar as notícias 
        function carregarNoticias() { 
            $.get("http://localhost:3000/noticias", function (data, status) { 
                                                
                $('#news').empty(); 
                $('#news').append('<h1>Listagem de Notícias</h1>'); 
                data.forEach(function (noticia) { 
                    $('#news').append( 
                        `<div id="noticia-${noticia.id}"> 
                            <h3>${noticia.titulo}</h3> 
                            <p>${noticia.conteudo}</p> 
                            <button onclick="editNews(${noticia.id})">Editar</button> 
                            <button onclick="deleteNews(${noticia.id})">Excluir</button> 
                        </div>` 
                    ); 
                }); 
            }); 
        } 
                                                
                                                 
        // Função para excluir uma notícia 
        window.deleteNews = function (id) { 
            $.ajax({ 
                url: `http://localhost:3000/noticias/${id}`, 
                type: 'DELETE', 
                success: function (result) { 
                    alert('Notícia excluída com sucesso!'); 
                    carregarNoticias(); 
                }, 
                                                
                error: function (err) { 
                    alert('Erro ao excluir a notícia.'); 
                } 
            }); 
        } 
                                                
        // Função para redirecionar para a página de edição 
        window.editNews = function (id) { 
            window.location.href = `editar.html?id=${id}`; 
        } 
                                                
        // Carregar as notícias ao carregar a página 
        carregarNoticias(); 
});

Quando o botão Excluir é clicado, a função deleteNew é chamada com o ID da notícia, que realiza uma requisição do tipo DELETE para o endpoint http://localhost:3000/noticias/${id}. Se a operação for bem-sucedida, é exibido um alerta de sucesso e recarrega a lista de notícias chamando a função carregarNoticias. Caso ocorra algum erro, será exibido um alerta.

Quando o botão Editar é clicado, a função editNews é chamada com o ID da notícia, que redirecionará o usuário para a página editar.html com o ID respectivo.

Passo 3: crie o arquivo editar.js para gerenciar a lógica de edição. Nesse arquivo, implemente o código para carregar os dados da notícia a ser editada e enviar as alterações ao servidor. 

                                         
$(document).ready(function() { 
    // Obter o ID da notícia a partir da URL 
    const urlParams = new URLSearchParams(window.location.search); 
    const noticiaId = urlParams.get('id'); 
                                                
    // Carregar os dados da notícia 
    $.get(`http://localhost:3000/noticias/${noticiaId}`, function(data, status) { 
        $('#titulo').val(data.titulo); 
        $('#conteudo').val(data.conteudo); 
    }); 
                                                                                             
    // Submeter o formulário de edição 
    $('#editForm').submit(function(event) { 
                                                
        // Impede o envio normal do formulário 
        event.preventDefault(); 
                                                
        // Serializa os valores do formulário 
        const formValues = { 
            titulo: $('#titulo').val(), 
            conteudo: $('#conteudo').val() 
        }; 
                                                
        $.ajax({ 
            url: `http://localhost:3000/noticias/${noticiaId}`, 
            type: 'PUT', 
            data: JSON.stringify(formValues), 
            contentType: 'application/json', 
            success: function(result) { 
            $('#result').text('Notícia atualizada com sucesso!'); 
        }, 
        error: function(err) { 
            alert('Erro ao atualizar a notícia.'); 
            } 
        }); 
    }); 
});
