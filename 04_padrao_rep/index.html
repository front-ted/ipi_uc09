Introdução

Ao desenvolver ou consumir APIs (application programming interface) REST (representational state transfer), é fundamental contar com ferramentas que facilitem os testes e a validação das requisições. O Postman é uma dessas ferramentas indispensáveis para desenvolvedores. Ele permite simular chamadas para APIs de forma intuitiva, ajudando a identificar e corrigir problemas rapidamente.

Para que você compreenda o funcionamento do Postman e sua aplicação, será explorado seu uso por meio de exemplos que ilustram a manipulação de uma API de gerenciamento de notícias, a mesma criada no material sobre programação de serviços. Ao longo deste guia, você aprenderá a criar requisições, enviar dados e interpretar as respostas do servidor.

O que é o Postman?

Antes de iniciar o uso prático do Postman, é importante relembrar que ferramentas como esta têm o objetivo de facilitar a interação com APIs, especialmente no contexto de desenvolvimento e testes. Como visto no material sobre protocolo HTTP e representação de dado, APIs REST utilizam métodos HTTP (hypertext transfer protocol) como GET, POST, PUT e DELETE para realizar operações sobre recursos. O Postman serve de interface gráfica para realizar essas operações de maneira organizada e eficiente.

Instalando o Postman

Antes de explorar as funcionalidades do Postman, é preciso garantir que ele esteja instalado corretamente no seu dispositivo. Lembre-se de que, em outros conteúdos, você viu como instalar e configurar ferramentas e o ambiente de desenvolvimento. Agora, veja como configurar a ferramenta de teste.

Para iniciar, você precisa instalar o Postman em seu dispositivo. Aqui estão os passos:

  1. Acesse o site oficial do Postman.
  2. Baixe a versão compatível com o seu sistema operacional (Windows, macOS ou Linux).
  3. Instale e abra o programa de acordo com o processo normal do seu sistema operacional.
  4. Faça login ou crie uma conta.

O Postman também oferece uma versão web, mas, para testar APIs locais, é recomendável usar a versão desktop.

Criando sua primeira requisição

Agora que você instalou o Postman, veja como criar sua primeira requisição para uma API REST. Antes disso, é importante se lembrar do conceito de endpoints e métodos HTTP. Um endpoint é o ponto de acesso a um recurso na API. Com base nele, são feitas as chamadas, enquanto os métodos (GET, POST, PUT, DELETE) definem a operação a ser realizada.

1. Para testar sua API, ela deve estar rodando. Então, abra a pasta da API criada no conteúdo de programação de serviços e levante com o comando: 
                                    
    node index.js
2. Abra o Postman e clique em Create a New Collection ou no botão + para criar uma nova aba.

Tela de criação de uma nova coleção no Postman com o botão + clicado e o cursor sobre REST API basics.

3. Escolha o método HTTP apropriado (GET, POST, PUT, DELETE).
4. Insira a URL do endpoint no campo de URL: http://localhost:3000/noticias

Adicionando cabeçalhos à requisição

Os cabeçalhos são partes fundamentais de uma requisição HTTP. Eles adicionam informações complementares que ajudam o servidor a processar as solicitações de forma adequada. Anteriormente, você viu como cabeçalhos como Content-Type influenciam a interpretação dos dados enviados e recebidos.

No Postman:

  1. Clique na aba Headers.
  2. Insira pares de chave e valor, como:
    3. - Content-Type: indica o formato dos dados enviados (exemplo: application/json).

Enviando dados no corpo da requisição

Requisições que utilizam métodos como POST ou PUT frequentemente exigem o envio de dados no corpo (body). É no body que são especificadas as informações que se quer armazenar ou atualizar no servidor. Esse conceito foi introduzido nos tópicos de manipulação de dados em APIs.

Para enviar dados no Postman:

  1. Clique na aba Body.
  2. Selecione a opção raw e escolha o formato JSON.
  3. Digite o conteúdo que será enviado. Por exemplo:
    4.                                 
    ``` 
    { 
        "titulo": "Evento local atrai milhares de visitantes", 
        "conteudo": "A festa da cidade reuniu mais de 10 mil pessoas neste fim de semana." 
    } 
    ```

Enviando a requisição

Agora que você configurou os detalhes da requisição, o próximo passo é enviá-la. Esse processo ativa a comunicação com o servidor e retorna os resultados para análise. Em outros conteúdos, discutiu-se o conceito de respostas HTTP e como interpretar os códigos de status, como 200 OK ou 404 Not Found.

Quando estiver pronto, clique no botão Send. O Postman mostrará:

  • Status da resposta: exemplo: 200 OK (sucesso) ou 404 Not Found (erro).
  • Cabeçalhos da resposta: metadados enviados pelo servidor.
  • Corpo da resposta: dados retornados pelo servidor.

Encerramento

Os serviços REST são a base da comunicação moderna entre sistemas. Compreender como implementá-los e testá-los com o Postman é essencial para desenvolvedores. A cada requisição realizada, você aprimora suas habilidades em diagnóstico e depuração, garantindo que suas APIs atendam aos requisitos funcionais e de desempenho.

Em outros conteúdos, você avançará para a etapa de consumir essas APIs utilizando jQuery. Você aprenderá como integrar o front-end com os serviços testados neste guia, criando aplicações mais completas e dinâmicas.
